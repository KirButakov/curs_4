# Сервис управления рассылками

Этот проект представляет собой веб-приложение для управления рассылками сообщений клиентам. Он разработан на Django и включает функциональность для создания, редактирования, удаления и отправки рассылок.

## Основные функции

- **Управление клиентами:**
  - Добавление, редактирование и удаление клиентов.
  - Поля: email, ФИО, комментарий.

- **Управление сообщениями:**
  - Добавление, редактирование и удаление сообщений.
  - Поля: тема письма, тело письма.

- **Управление рассылками:**
  - Добавление, редактирование и удаление рассылок.
  - Поля: дата и время начала, дата и время окончания, статус, сообщение, клиенты.
  - Статусы: "Создана", "Запущена", "Завершена".

- **Попытки рассылок:**
  - Фиксация каждой попытки отправки.
  - Поля: дата и время попытки, статус, ответ почтового сервера.

- **Аутентификация:**
  - Регистрация, вход, выход.
  - Редактирование профиля (email, аватар, номер телефона, страна).

- **Главная страница:**
  - Отображает статистику: количество рассылок, активных рассылок и уникальных клиентов.

## Установка

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/KirButakov/curs_4
   
2. Установите зависимости:

pip install -r requirements.txt

3. Настройте переменные окружения: 

Создайте файл .env в корне проекта и добавьте туда:

DB_NAME=job_search

DB_USER=xxxxx

DB_PASSWORD=xxxxx

DB_HOST=localhost

DB_PORT=5432

DEBUG=True 

4. Примените миграции: 

python manage.py makemigrations

python manage.py migrate

5. Запустите сервер:

python manage.py runserver

## Использование

 - Главная страница: Отображает статистику по рассылкам.

 - Клиенты: Управление клиентами (добавление, редактирование, удаление).

 - Сообщения: Управление сообщениями (добавление, редактирование, удаление).

 - Рассылки: Управление рассылками (добавление, редактирование, удаление, запуск).

 - Попытки рассылок: Просмотр попыток отправки рассылок.

 - Профиль: Редактирование профиля пользователя.

## Технические детали

 - Язык программирования: Python 3.12

 - Фреймворк: Django 4.2

 - База данных:PostgreSQL.

 - Статические файлы: Используется django.contrib.staticfiles.

 - Аутентификация: Кастомная модель пользователя (users.User).
